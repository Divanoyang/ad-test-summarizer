# 自动驾驶测试问题分类指南

本文件是分类判断的参考依据。所有问题归入四大类（按输出顺序）：主观驾驶感受、高风险问题、高频问题、其他问题。

---

## 👤 类别一：主观驾驶感受

**定义**：驾驶员或乘客基于个人体验提出的主观反馈，包括舒适性、安心感、体感异常，以及影响体验的功能稳定性问题。

**典型主题分组**：

### 纵向体感
- 过度礼让与蠕行：对静止VRU、横穿行人/车辆过度礼让，频繁刹停或低速蠕行
- 加速逻辑异常：猛加速带来压迫感；跟车起步/转弯后加速过慢；体感割裂
- 误刹与点刹：对无切入意图车辆误触发急刹；旁车预测误入侵导致误降速；路口无障碍不合理减速

### 横向控制感受
- 方向盘晃动/甩动：变道犹豫Path来回跳导致晃动；路口转弯时方向盘甩动、抽搐
- 绕行压迫感：窄路速度快距离障碍物近；绕行幅度不足；借道绕行时车身甩动
- 弯道入侵感受：弯道不居中；入侵对向车道带来压迫感

### 功能稳定性
- 功能异常退出/Crash/卡死
- 导航降级难恢复（如删除途经点后NOA无法恢复）

**判断要点**：关键词包括"体感"、"压迫感"、"安心感"、"恐慌"、"舒适"、"割裂"、"晃动"等主观感受词汇。功能Crash/异常退出也归入此类。

---

## 🚨 类别二：高风险问题

**定义**：涉及碰撞风险、车辆失控、严重交通违规或博弈失败的安全隐患。

**典型主题分组**：

### 碰撞与失控风险
- 弯道失控：方向失控甩向对向车道
- 借道绕行冲突：绕行时遇来车横向距离过近
- 起步/跟随风险：异常绕行旁有并行车；跟车刹不住；盲区目标驶出未刹车；前车倒车不避让
- 路口冲突：转弯挤压来车；不避让主路直行；车道减少抢道
- 绕行避让不及时

### 严重违规
- 红绿灯误检导致闯红灯
- 斑马线行人不礼让
- 前车漏检有追尾风险

### 博弈失败
- Cut-in无合理避让
- 变道不顾后方来车强行切入
- 拥堵时无视后方来车强行绕行

**判断要点**：关键词包括"碰撞"、"失控"、"甩"、"闯红灯"、"不礼让"、"追尾"、"Cut-in"、"强行"、"挤压"、"刹不住"等。只要有安全隐患就归入此类。

---

## 🔁 类别三：高频问题

**定义**：在本次提交中多次/多车出现的同类问题，反映系统性缺陷而非偶发。

**典型主题分组**：

### 横向规划与控制
- 不合理变道：来回变道、变道犹豫骑线、无效变道
- 压线行驶：选道不及时压实线；方向盘持续偏右压线；不居中
- 轨迹异常：Path选入非机动车道；转弯上马路牙子；停车不居中

### 纵向控制策略
- 无故刹停：路口/绿灯/窄路莫名刹停；误检待转区越线刹停；避让后不起步
- 误检点刹：绿闪黄误检停止线；被遮挡红绿灯误检信号；对横穿目标二次点刹
- 感知误判导致的减速：误判大车压线急点刹；ILQR不合理急刹
- 特定车型问题：某车型特有的高频点刹等

### 特定场景逻辑失效
- 掉头路口行驶一半刹停
- LaneClose识别不稳定
- 特殊信号灯场景不起步

**判断要点**：同一现象（变道、压线、点刹、刹停等）在问题列表中出现 2 次及以上，或明确标注为多次/多车出现。重点看是否反映系统性缺陷。

---

## ℹ️ 类别四：其他问题

**定义**：不属于以上三类的问题，通常是感知渲染错误、导航系统故障、限速偏差等。

**典型主题分组**：

### 感知与模型错误
- 渲染错误：目标物识别类型错误（如货车识别为公交车、三轮车渲染成二轮车等）
- 数据丢失：近处车辆/目标丢失
- 车型误识

### 导航与系统故障
- 导航选路错误
- 地图匹配报错（注意：不要写"高精地图"，使用"导航地图"）
- 信号灯绑定错误

### 其他
- 限速偏差
- 日志/数据问题
- 非功能性建议

---

## 分类判断流程

```
拿到一条问题
   │
   ▼
涉及碰撞/失控/违规/博弈安全隐患？──是──→ 🚨 高风险问题
   │
   否
   │
   ▼
同类问题出现≥2次或属于系统性缺陷？──是──→ 🔁 高频问题
   │
   否
   │
   ▼
涉及主观体感/舒适性/功能Crash？──是──→ 👤 主观驾驶感受
   │
   否
   │
   ▼
ℹ️ 其他问题
```

**核心原则**：
1. 安全最高：有安全隐患 → 一律高风险
2. 系统性问题优先：多次出现反映系统缺陷 → 高频
3. 主观体验单独归类：体感、安心感、功能稳定性 → 主观驾驶感受
4. 兜底：以上都不是 → 其他

**输出顺序**：主观驾驶感受(1) → 高风险问题(2) → 高频问题(3) → 其他问题(4)

**聚类要求**：
- 分类完成后，必须对每类中的问题进行聚类提炼
- 不能直接复制用户原文，需要用专业语言重新概括
- 相似问题合并为一条，用分号分隔不同表现形式
